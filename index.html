Website

Written on 30 Nov 2020

Github repository

Intro

This is the 3rd iteration of my personal website as part of my yearly website refresh.
This time around, the design is inspired by:

  • The opening of the 3rd arc of the Owarimonogatari tv series.
  • The opening of the 1st Persona 2 game.
  • The UI of the Persona 5 game.
  • The Cowardly Witch's website.

Tech Stack

EJS & Markdown is used for layout & content.
Sass is used for styling.

JavaScript libraries powered by NodeJS are used to compile source code & optimize assets.
Gulp is used to manage the aforementioned build process.

Performance

The entire website inclusive of all resources for my 1080p screen has a transfer size of only 223kB, of which:

Type Approx. Size
WEBP 70kB (31%)
SVG 84kB (38%)
WOFF2 32kB (15%)
PNG 22kB (10%)
CSS 8kB (3%)
HTML 7kB (3%)
JS 200 Bytes

For comparison, the Google homepage trasfers a total of 458kB on the same device.

During rendering performance testing, the website is able to both play animations & scroll smoothly on a 2016 Asus Zenfone 3 & 2015 Macbook Pro.
The test is done in Firefox & Chrome for the former and Safari for the latter.

This website has a Lighthose score of 100/100 in the Performance category.

Compatibility

This website is compatible with:

  • Firefox
  • Chrome
  • Edge
  • Safari
  • Internet Explorer 11

It also functions fine with JavaScript disabled.
The only line of JavaScript used is just to make sure the opening animation doesn't play if the page loads with a valid hash fragment.

This website supports screens of:

  • Size as small as 240 320.
  • Aspect-ratio as wide as 21:9.
  • Aspect-ratio as tall as 9:21.

Accessibility

This website has the following accessibility features:

  • Keyboard-friendly.
  • NVDA screen reader friendly.
  • Has text contrast ratio of 4.5:1 and above.
  • Support text zoom level up to 200%.
  • Atkinson Hyperlegible font for low-vision readers.
  • Valid HTML & CSS verified by the W3C Markup Validation Service.

This website has a Lighthouse score of 100/100 in the Accessibility category.

Privacy

This website does not collect any information about you at all.

Zip, zilch, nada. No trackers, no analytics.

Digital Rain

Github repository
Link to Demo

High CPU/GPU usage in demo.

Background

The "Matrix Digital Rain" refers to the iconic falling green code sequence featured in the opening of the Matrix series.

Here is a video clip from one of the movies showing the Matrix Digital Rain: Download MKV File (4.8MB).

Motivation

Lots of people already attempted to reproduce the Matrix Digital Rain in code, but I found their results to be noticably different to the real thing and unsatisfying. So I decided to take a stab at it to see what the challenges are.

Observations

Looking at the video clip above, here are some of my observations on what's going on:

  • The code is katakana, numbers and some symbols inverted on the Y axis.
  • The code doesn't actually move but continuously appended at the bottom.
  • At the end of the trail, the code fades out.
  • There seem to be some sort of motion blur on the code fading out.
  • There can be more than 1 stream of code per column.
  • Each frame, some of the characters change but most remain the same.
  • The glow is not constant but pulsates.
  • The glow is independent of the characters.
  • The light of the glow bleeds into nearby code.

Conclusion

Looking at my list of observations, it's no wonder that making a convincing replica in code is difficult.

After spending a lazy weekend fiddling around, I would say for most practical purposes, it is better to just take the idea of the digital rain and make it beautiful rather than try to mimic it faithfully.
